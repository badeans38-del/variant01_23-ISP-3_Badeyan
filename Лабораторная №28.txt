 Тема задания: «Основы протокола HTTP: структура запросов, статус-коды и методы взаимодействия»
 Цель задания:
 Разобрать принципы работы протокола HTTP, изучить структуру HTTPзапросов, статус-коды, различия между методами GET и POST, а также понять развитие протокола от HTTP/1.0 к HTTP/1.1.
 В ходе выполнения задания необходимо научиться:
 ● Ответить на вопросы о структуре HTTP-запроса
 ● Объяснить назначение и диапазоны статус-кодов
 ● Описать различия между методами GET и POST
 ● Пояснить почему HTTP работает поверх TCP
 ● Рассмотреть изменения, появившиеся в HTTP/1.1 по сравнению с HTTP/1.0
Задание: Ответьте на следующие вопросы:
1. Из каких частей состоит HTTP-запрос (на уровне текста)?
 Стартовая строка (Start Line): Метод (GET/POST), путь (URI) и версия протокола.
 Заголовки (Headers): Метаданные (например, Host, User-Agent, Content-Type).
 Пустая строка: Разделитель.
 Тело (Body): (Опционально) Данные запроса (например, в POST).

2. Что такое статус-код в HTTP и какие значения он может иметь?
Это трехзначное число, сообщающее результат обработки запроса сервером.
 1xx: Информационные.
 2xx: Успех (200 OK).
 3xx: Перенаправление (301 Moved Permanently).
 4xx: Ошибка клиента (404 Not Found, 403 Forbidden).
 5xx: Ошибка сервера (500 Internal Server Error).



3. Чем отличаются методы GET и POST?
 GET: используется для получения данных. Параметры передаются в URL (видно всем). Идемпотентен (повтор запроса не меняет состояние сервера). Тела запроса обычно нет.
 POST: используется для отправки данных (создания ресурса). Данные в теле запроса (скрыты из адресной строки). Не идемпотентен.

4. Почему говорят, что HTTP работает поверх TCP?
HTTP является протоколом прикладного уровня, которому требуется надежная передача текста (HTML, JSON) и файлов. Он делегирует задачу гарантии доставки, целостности и порядка данных протоколу транспортного уровня — TCP. Если бы HTTP работал поверх UDP, страницы загружались бы с "битыми" кусками текста.

5. Что изменилось в HTTP/1.1 по сравнению с HTTP/1.0?
 Keep-Alive: Соединение TCP не закрывается после каждого запроса, позволяя загружать много ресурсов (картинки, скрипты) через одно соединение.
 Host Header: Обязательное требование указывать домен (позволяет виртуальный хостинг).
 Chunked transfer encoding: Возможность передавать данные частями.
